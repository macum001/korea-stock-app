# TESTING_CHECKLIST

## 장중 실시간 테스트
1. 정규장 09:00~15:30에 삼성전자/하이닉스 등 거래량 많은 종목 상세 진입.
2. 현재가, 체결창, 차트 마지막 캔들의 close가 같은 체결 tick 기준으로 움직이는지 확인.
3. 호가창이 100~300ms 단위로 부드럽게 바뀌는지 확인.
4. 체결 리스트가 300건까지 유지되고 스크롤이 끊기지 않는지 확인.

## WebSocket 재연결 테스트
1. 종목 상세 화면 진입.
2. 브라우저 네트워크를 Offline으로 전환 후 10~20초 대기.
3. Online으로 복구.
4. 마지막 호가/체결이 지워지지 않고 재연결 후 신규 tick이 이어지는지 확인.

## 장마감 테스트
1. 15:30 이후 종목 상세 진입.
2. 호가창이 “데이터 없음”이 아니라 종가/마지막 snapshot 기준 안내를 표시하는지 확인.
3. `/api/market/snapshot/:code`로 snapshot이 조회되는지 확인.
4. 필요 시 `POST /api/market/snapshot/:code/finalize`로 강제 확정.

## 차트 테스트
1. 1/3/5/10/15/30/60/120/240분봉을 각각 선택.
2. 한국장 09:00 기준으로 bucket이 나뉘는지 확인.
3. 새 체결이 같은 bucket이면 마지막 캔들의 high/low/close/volume만 바뀌는지 확인.
4. 새 bucket이면 새 캔들이 추가되는지 확인.

## 10년치 일봉 테스트
1. 일봉/주봉/월봉/년봉 선택.
2. 10년치 일봉 데이터가 과도하게 오래 걸리지 않는지 확인.
3. 차트 진입 전 앱 첫 로딩 bundle이 무거워지지 않았는지 확인.

## 투자자별 수급 테스트
1. 20일/60일/1년 전환.
2. 개인/외국인/기관 순매수 수량이 정상 표시되는지 확인.
3. 5일/20일 누적 값이 일별 합산과 일치하는지 확인.
4. dataStatus가 ESTIMATED/DELAYED/CONFIRMED로 구분되는지 확인.
